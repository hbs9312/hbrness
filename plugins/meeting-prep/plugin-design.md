# 회의 준비 플러그인 설계서

---

## 1. 컴포넌트 리스트

| # | 이름 | 유형 | 역할 | 변동성 |
|---|------|------|------|--------|
| 1 | **spec-scanner** | 스킬 | 기획서 구조 파악 + 불명확 항목 탐지 | 높음 (기획서 구조 변경 대응) |
| 2 | **impl-scanner** | 스킬 | 구현 프로젝트 구조 + 현황 분석 | 높음 (기술스택 변경 대응) |
| 3 | **meeting-doc-gen** | 스킬 | 중간 포맷 → 고정 템플릿 회의 문서 변환 | 낮음 (템플릿 고정) |
| 4 | **meeting-orchestrator** | 에이전트 | 스캐너 병렬 실행 + 결과 종합 + 문서 생성 호출 | 낮음 (플로우 고정) |

---

## 2. 사용 플로우

```
사용자: "다음 회의 준비해줘" / "채팅 기능 관련 회의 안건 정리해줘"
         │
         ▼
┌─────────────────────────────────────────────────────────────┐
│  meeting-orchestrator (에이전트 — 진입점)                     │
│                                                              │
│  Step 1. 사용자에게 확인                                      │
│     - 회의 주제/범위, 경로, 회의 유형                          │
│                                                              │
│  Step 2. 서브에이전트 병렬 spawn                              │
│     ┌──────────────────┐    ┌──────────────────┐            │
│     │ Sub-Agent A       │    │ Sub-Agent B       │            │
│     │ spec-scanner 실행 │    │ impl-scanner 실행 │            │
│     │ → spec-analysis   │    │ → impl-analysis   │            │
│     └────────┬─────────┘    └────────┬─────────┘            │
│              └──────────┬───────────┘                        │
│                         ▼                                    │
│  Step 3. 중간 결과 수집 + 교차 분석                           │
│                         │                                    │
│                         ▼                                    │
│  Step 4. meeting-doc-gen 스킬 실행                           │
│     → 고정 템플릿으로 최종 문서 생성                           │
│     → meeting-prep/{주제}_{날짜}.md                          │
└─────────────────────────────────────────────────────────────┘
```

### 2.1 플로우 상세

**Step 1 — 범위 확인** (meeting-orchestrator)
에이전트가 사용자에게 회의 주제와 범위를 묻는다.

**Step 2 — 병렬 분석** (서브에이전트 × 2)
- Sub-Agent A: spec-scanner 스킬 지침에 따라 docs/ 스캔 → spec-analysis.md 생성
- Sub-Agent B: impl-scanner 스킬 지침에 따라 프로젝트 스캔 → impl-analysis.md 생성
- **독립 컨텍스트**에서 병렬 실행되므로 각각 깊이 있는 분석 가능

**Step 3 — 교차 분석** (meeting-orchestrator)
두 중간 포맷을 읽어 기획↔구현 갭 분석, 우선순위 산정

**Step 4 — 문서 생성** (meeting-doc-gen 스킬)
교차 분석 결과와 중간 포맷을 고정 템플릿에 적용하여 최종 문서 생성

---

## 3. 중간 포맷 (Inter-Skill Protocol)

스캐너 스킬들이 출력하고 meeting-doc-gen이 소비하는 중간 포맷을 정의한다. 이 포맷이 스킬 간 계약(contract) 역할을 한다.

### 3.1 spec-analysis.md 포맷

```markdown
# 기획서 분석 결과
> 분석일: {날짜}
> 스캔 범위: {전체 | 특정 도메인}
> 스캔 경로: {docs 경로}

## 구조 요약
- 전체 도메인 목록과 문서 수
- 각 도메인의 핵심 기능 개요

## 불명확 항목
### [{도메인}] {항목 제목}
- **파일**: {파일 경로}:{행 번호}
- **현황**: 현재 어떻게 기술되어 있는지
- **유형**: 미정 | 미구현 | 참조깨짐 | 정책미확정 | 빈값 | 모호한정의
- **상세**: 왜 불명확한지 설명
- **영향 범위**: 이 항목이 영향을 미치는 다른 기능/문서

(항목 반복)

## 문서 간 참조 현황
- 깨진 참조 목록
- 순환 참조 여부

## 최근 변경 사항
- 최근 수정된 기획서 파일들 (git log 기반)
```

### 3.2 impl-analysis.md 포맷

```markdown
# 구현 현황 분석 결과
> 분석일: {날짜}
> 분석 대상: {프로젝트 목록}

## [{프로젝트명}] 구조 요약
- 기술 스택
- 디렉토리 구조 개요
- 주요 모듈/기능 목록

## [{프로젝트명}] 구현 상태
### 구현 완료 기능
| 기능 | 관련 파일 | 비고 |
|------|----------|------|

### 부분 구현 / 진행 중 기능
| 기능 | 관련 파일 | 현재 상태 | 비고 |
|------|----------|----------|------|

### 미구현 기능 (기획서 대비)
| 기획서 기능 | 기획서 경로 | 구현 상태 | 비고 |
|------------|-----------|----------|------|

## [{프로젝트명}] 최근 작업 동향
- 최근 커밋 요약 (git log)
- 최근 마이그레이션/스키마 변경 (해당 시)

(프로젝트별 반복)
```

## 4. 최종 출력 템플릿 (meeting-doc-gen)

```markdown
# {회의 주제} ({날짜})

> 회의 목적: {한 줄 요약}

---

## 1. 기획 검토 안건

### 1.1 정책 결정 필요 항목
| # | 항목 | 현황 | 결정 필요 사항 | 관련 파일 |
|---|------|------|--------------|----------|

### 1.2 명세 불명확 항목
| # | 항목 | 현황 | 구체화 필요 사항 | 관련 파일 |
|---|------|------|----------------|----------|

### 1.3 문서 정합성 이슈
- 깨진 참조, 모순되는 정의 등

---

## 2. 구현 현황 리포트

### 2.1 최근 진행 상황
- 프로젝트별 최근 작업 요약

### 2.2 기획 대비 구현 갭
| # | 기획 기능 | 기획서 | 구현 상태 | 비고 |
|---|----------|--------|----------|------|

### 2.3 기술적 논의 필요 사항
- 구현 중 발견된 기술적 이슈, 설계 결정 필요 사항

---

## 3. 종합 안건 (우선순위순)

### 🔴 긴급 (블로커)
- 개발 진행을 막고 있는 미결정 사항

### 🟡 중요 (이번 스프린트)
- 가까운 시일 내 결정/처리해야 할 사항

### 🟢 참고 (백로그)
- 인지해두면 좋은 사항

---

## 4. 회의 시 결정 체크리스트

- [ ] {결정 항목 1}
- [ ] {결정 항목 2}
...
```

## 5. 플러그인 디렉토리 구조

```
meeting-prep/
├── .claude-plugin/
│   └── plugin.json                  # 플러그인 매니페스트
├── agents/
│   └── meeting-orchestrator/
│       └── AGENT.md                 # 오케스트레이터 에이전트
├── skills/
│   ├── spec-scanner/
│   │   └── SKILL.md                 # 기획서 분석 스킬
│   ├── impl-scanner/
│   │   └── SKILL.md                 # 구현 현황 분석 스킬
│   └── meeting-doc-gen/
│       ├── SKILL.md                 # 문서 생성 스킬
│       └── references/
│           ├── output-template.md   # 최종 출력 고정 템플릿
│           └── inter-skill-protocol.md  # 중간 포맷 정의
└── README.md
```
