# API 클라이언트 동기화 (F-API)

ultrathink

당신은 프론트엔드 DX(Developer Experience) 엔지니어입니다.
`openapi/openapi.yaml` 로부터 TS API 클라이언트 함수·타입 정의·MSW 핸들러를 codegen 합니다. `info.version` + `contractHash` 이중 박제로 stale 검출을 보장하는 것이 이 스킬의 핵심 책임입니다.

## 컨텍스트 로드

- **프로젝트 설정**: [frontend.md](../../context/frontend.md) — `api_contract` 섹션 **필수**, `api_client`, `server_state`
- **태스크-파일 맵**: `.frontflow/task-file-map.md` (있으면)

## 입력

1. **OpenAPI 문서**: `frontend.md.api_contract.source` 에서 경로 또는 URL 로드
   - 모노레포 상대경로 예: `openapi/openapi.yaml`
   - URL 예: `https://api.example.com/openapi.json` (fetch + 로컬 캐시 `~/.hbrness/api-contract/{hash}.yaml`)
2. **인자 override**: `$ARGUMENTS` 로 직접 경로 지정 시 `frontend.md.api_contract.source` 보다 우선

## ★ 실행 위치 — impl-error-handling 후, impl-api-integration 전 ★

이 스킬은 **impl-error-handling 이후** 실행되어야 함 — 에러 코드 계약이 확정된 후.
**impl-api-integration 보다 먼저** — codegen 결과를 import 하기 때문.

실행 순서:
```
impl-error-handling       ← TS §에러 코드 맵 (+ UI ui_flow 힌트)
     │
     ▼
sync-api-client           ← openapi.yaml → codegen (types + client + MSW)   ★Phase 1 (3)
     │
     ▼
impl-interactions         ← FS BR + UI 인터랙션 + WF 상태 매트릭스 + TS 상태 enum
     │
     ▼
impl-api-integration      ← codegen import + 비즈니스 와이어링 (stub 교체)
```

## ★ 핵심 동작 ★

### 1. OpenAPI 로드

- `frontend.md.api_contract.source` (또는 `$ARGUMENTS`) 에서 `openapi.yaml` 파싱
- `info.version` 추출 (계약 표면 버전)
- `x-contract-hash` (contractHash) 추출 — `backflow:export-api-contract` 가 박제한 sha256

### 2. Lock 비교

- `.frontflow/api-contract.lock` 가 존재하면 `version` / `contract_hash` 비교
- **변동 없으면**: no-op + info 메시지 ("openapi.yaml 변경 없음 — codegen 스킵")
- **변동 있으면**: 아래 3~5 단계 전체 실행

### 3. Codegen

`frontend.md.api_contract.generator` 설정에 따라 실행. 출력:

- **`{client_dir}/{tag}.ts`** — operationId 기반 함수 (tags 그룹핑)
  - 예: `src/api/generated/speakers.ts`, `src/api/generated/workspaces.ts`
- **`{client_dir}/index.ts`** — barrel export (모든 tag 파일 re-export)
- **`{types_file}`** (예: `src/api/types.gen.ts`) — `components.schemas` → TypeScript types (ErrorCode union 포함)
- (`emit_msw=true`) **`{msw_handlers_file}`** — MSW handlers from examples (`src/mocks/handlers.gen.ts`)
- (`emit_query_keys=true`) **queryKey 상수** — generator 지원 시 각 tag 파일 내 포함

Generator 옵션별 지원 범위:

| generator | Phase 1 등급 | 출력 범위 | 비고 |
|---|---|---|---|
| **`openapi-typescript-codegen`** | **first-class (default)** | types + 함수 + axios/fetch + queryKey | 단일 안정 지원 |
| `orval` | experimental | types + 함수 + tanstack-query 훅 + MSW | 옵션 많음 — Phase 2 stable |
| `openapi-zod-client` | experimental | types + 함수 + zod 런타임 검증 | 빌드 사이즈 ↑ — Phase 2 |
| `manual` | minimal | **types + fetch 함수만** (MSW/queryKey 미지원) | external 의존 회피용 |

`emit_msw: true` + 미지원 generator 조합: warning 출력 + 옵션 무시.

### 4. Lock 갱신

`.frontflow/api-contract.lock` JSON 작성 (commit 대상):

```json
{
  "version": "{info.version}",
  "contract_hash": "{x-contract-hash}",
  "source_etag": "{ETag 헤더값 또는 파일 mtime hash}",
  "generated_at": "{ISO8601 타임스탬프}",
  "generator": "{frontend.md.api_contract.generator}",
  "generator_version": "{설치된 generator 버전}"
}
```

### 5. 헤더 박제

모든 generated 파일 최상단에 AUTO-GENERATED 주석 박제:

```typescript
// AUTO-GENERATED by frontflow:sync-api-client
// from openapi/openapi.yaml@info.version={version} contractHash={hash}
// Re-run the skill to regenerate. Manual edits trigger validate-code critical.
// generator: {generator}
```

## 출력 구조

```
src/api/generated/
  ├── speakers.ts          # tag=speakers 의 모든 operationId
  ├── workspaces.ts
  └── index.ts             # barrel export
src/api/
  └── types.gen.ts         # components.schemas → TypeScript types (ErrorCode enum 포함)
src/mocks/
  └── handlers.gen.ts      # MSW handlers (emit_msw=true 시)
.frontflow/
  └── api-contract.lock    # { version, contract_hash, source_etag, generated_at, generator } JSON
```

**사람 작성 파일** (이 스킬이 건드리지 않음):
- `src/api/client.ts` — fetch/axios/ky 인스턴스 (impl-api-integration 책임)
- `src/mocks/server.ts` — MSW 서버 설정 (사람 작성)

## 출력 예시

### `src/api/generated/speakers.ts`

```typescript
// AUTO-GENERATED by frontflow:sync-api-client
// from openapi/openapi.yaml@info.version=0.3.0 contractHash=7f3b2a1c8d5e...
// Re-run the skill to regenerate. Manual edits trigger validate-code critical.
// generator: openapi-typescript-codegen

import { client } from '../client';
import type { EnrollRequest, EnrollResponse } from '../types.gen';

export async function enrollSpeaker(body: EnrollRequest): Promise<EnrollResponse> {
  const formData = new FormData();
  formData.append('name', body.name);
  formData.append('audio_file', body.audio_file);
  formData.append('workspace_id', body.workspace_id);
  return client.post('/speakers/enroll', formData);
}

export async function listSpeakers(workspaceId: string): Promise<SpeakerListResponse> {
  return client.get('/speakers', { params: { workspace_id: workspaceId } });
}

// Query keys (emit_query_keys=true)
export const speakerKeys = {
  all: ['speakers'] as const,
  list: (workspaceId: string) => [...speakerKeys.all, 'list', workspaceId] as const,
  detail: (id: string) => [...speakerKeys.all, 'detail', id] as const,
};
```

### `src/api/types.gen.ts`

```typescript
// AUTO-GENERATED by frontflow:sync-api-client
// from openapi/openapi.yaml@info.version=0.3.0 contractHash=7f3b2a1c8d5e...

export interface EnrollRequest {
  name: string;
  audio_file: File;
  workspace_id: string;
}

export interface EnrollResponse {
  speaker_id: string;
  name: string;
  embedding_status: EmbeddingStatus;
  created_at: string;
}

export type EmbeddingStatus = 'pending' | 'processing' | 'ready' | 'failed';

// XR-003: ErrorCode union 은 OpenAPI components.schemas.ErrorCode.enum 과 1:1
export type ErrorCode =
  | 'AUDIO_TOO_SHORT'
  | 'SPEAKER_DUPLICATE_NAME'
  | 'AUTH_INVALID_CREDENTIALS'
  | 'AUTH_TOKEN_EXPIRED'
  | 'SYSTEM_INTERNAL_ERROR';

export interface ErrorEnvelope {
  error: {
    code: ErrorCode;
    message: string;
    meta?: Record<string, unknown>;
  };
}
```

## Grace period (v1.x)

`openapi.yaml` 이 없거나 `api_contract.source` 미설정 시:
1. 경고: "frontend.md.api_contract.source 가 없습니다. backflow:export-api-contract 를 먼저 실행 권장."
2. `$ARGUMENTS` 로 경로 직접 지정 시도 가능 — 파일이 있으면 계속 진행
3. 파일도 없으면 codegen 전체 스킵 + 사용자에게 경로 지정 요청

## 멱등성

- 모든 generated 파일 상단 `// AUTO-GENERATED` 주석
- 재실행 시 lock 비교 → 변동 없으면 no-op
- `--force` 인자 시 lock 무시하고 강제 재생성

## 품질 자가 점검

- [ ] generated 파일이 openapi.yaml 의 모든 operationId 커버
- [ ] `types.gen.ts` 에 ErrorCode union 포함 (openapi.yaml.components.schemas.ErrorCode.enum 과 1:1)
- [ ] 모든 generated 파일에 AUTO-GENERATED 주석 + version/hash 헤더 박제
- [ ] `.frontflow/api-contract.lock` 생성/갱신 (version/contract_hash/source_etag/generated_at/generator/generator_version)
- [ ] `{client_dir}/index.ts` barrel export 생성
- [ ] (`emit_msw=true`) `handlers.gen.ts` 가 openapi.yaml 의 모든 path×method 커버
- [ ] (`emit_query_keys=true`) queryKey 상수 생성 (generator 지원 시)
- [ ] 미지원 generator + emit_msw/emit_query_keys 조합 시 warning 출력
- [ ] `src/api/client.ts` 등 사람 작성 파일은 수정하지 않음
