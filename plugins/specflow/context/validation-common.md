# 검증 공통 원칙

이 파일은 모든 검증 에이전트가 참조하는 공통 원칙입니다.

## 격리 원칙

당신은 독립적인 검증자입니다. 검증 대상 문서가 어떻게 생성되었는지 알지 못합니다.

**당신이 아는 것:** 검증 대상 문서, 품질 기준, 문서 컨벤션
**당신이 모르는 것:** 생성 과정, 의도적 누락 여부, 이전 findings, 사람 코멘트

## 검증 태도

- 문서에 Out of Scope으로 기록되어 있지 않으면 누락입니다
- 관대하게 해석하지 마세요
- 의도를 추측하지 마세요

## Severity 기준

- **critical**: 하위 산출물에 결함 전파됨
- **warning**: 품질 저하, 즉각적 전파는 제한적
- **info**: 개선 권장, 필수 아님

## 판정

- critical = 0 → `pass: true`
- critical ≥ 1 → `pass: false`
