# 문서 컨벤션

이 파일은 모든 specflow 스킬이 참조하는 공통 규칙입니다.
프로젝트별로 수정하지 마세요. 프로젝트별 용어는 `glossary.md`에 정의합니다.

## 문서 ID 체계

| 문서 유형 | 접두사 | 형식 | 예시 |
|-----------|--------|------|------|
| 기능 명세서 | FS | FS-{YYYY}-{NNN} | FS-2026-001 |
| 와이어프레임 | WF | WF-{YYYY}-{NNN} | WF-2026-001 |
| 기술 명세서 | TS | TS-{YYYY}-{NNN} | TS-2026-001 |
| 화면설계서 | UI | UI-{YYYY}-{NNN} | UI-2026-001 |
| 테스트 명세서 | QA | QA-{YYYY}-{NNN} | QA-2026-001 |

## 문서 상태값

문서 상태: `Draft` → `In Review` → `Approved` → `Deprecated`

## 메타데이터 필수 필드

모든 문서의 헤더에 반드시 포함:

```yaml
문서 ID: {접두사}-{YYYY}-{NNN}
작성일: {YYYY-MM-DD}
상태: {Draft | In Review | Approved | Deprecated}
작성자: {역할명}
리뷰어: {역할명 목록}
```

참조 문서가 있는 경우 추가:

```yaml
기능 참조: FS-{YYYY}-{NNN}
기술 참조: TS-{YYYY}-{NNN}
```

## 출력 형식 규칙

- 모든 산출물은 YAML 코드블록 안에 작성
- 섹션 구분은 `# ─────` 코멘트 라인 사용
- 참조 ID는 반드시 `{접두사}-{NNN}` 형식으로 인라인 표기
  예: "화자명 중복 불가 (BR-003)"

## 크로스 참조 규칙

명세 내 다른 지점을 가리킬 때는 **반드시 정확한 ID로 참조**한다. 위치 기반 대명사("위 조건", "아래 규칙", "상기 항목", "해당 건" 등)로 **다른 BR/AC/US/섹션**을 가리키면 안 된다. 위치 대명사는 문서가 리팩터링·재정렬될 때 깨지고, 독자가 스크롤해서 맥락을 찾아야 하므로 금지한다.

### 금지 예시

```
❌ BR-002: 위 조건에 해당하지 않는 콘텐츠는 자동 승인된다.
❌ BR-009: 상기 규칙 외의 상태 전이는 허용하지 않는다.
❌ AC-012: 해당 항목 미선택 시 에러를 표시한다.
```

### 권장 예시

```
✅ BR-002: BR-001의 검열 조건에 해당하지 않는 콘텐츠는 자동 승인된다.
✅ BR-009: BR-009에 명시된 전이 외의 상태 전이는 허용하지 않는다.  # 자기 참조도 ID 명시
✅ AC-012: 문제 항목(BR-010) 미선택 시 에러를 표시한다.
```

### 허용 예외: 자기 bullet 목록 지시

같은 BR/AC가 **자기 내부의 bullet 목록**을 즉시 뒤·앞에서 가리키는 경우에만 "아래 항목" / "위 항목"을 허용한다. 독자가 스크롤 없이 같은 블록 안에서 인접 bullet을 찾을 수 있을 때만 해당.

```
✅ BR-001: 아래 검열 조건에 1개 이상 해당하면 검수 대기로 노출한다.
    - 썸네일: "성인콘텐츠 가능성있음" OR ...
    - 내부 상황이미지: "성인콘텐츠 높음" OR ...
```

단, 이 BR-001을 **다른 위치(BR-002, AC-xxx 등)에서** 참조할 때는 반드시 `BR-001` ID로.
