# 화면설계서 추출 (G4)

이 스킬은 "생성"이 아니라 "추출 + 정규화"입니다.
디자이너의 산출물을 입력으로 받아 구조화된 문서로 변환합니다.

## 공통 컨텍스트 로드

- [conventions.md](../../context/conventions.md)
- [glossary.md](../../context/glossary.md)

## 입력

1. 와이어프레임(WF) 경로
2. 디자이너 산출물 (Figma 설명, 디자인 시스템 파일 등)

## 모드 판별

| 조건 | 모드 | 동작 |
|------|------|------|
| `--base` 인자 없음 | **full** (기본) | 전체 UI 명세를 새로 추출 |
| `--base [기존 UI 경로]` 있음 | **incremental** | 기존 UI를 보존하고 새로 추가된 WF 화면에 대한 UI 명세만 추출하여 병합 |

### incremental 모드 규칙

1. 기존 UI 파일을 Read로 읽고 현재 화면별 컴포넌트 목록 파악
2. WF에서 기존 UI에 매핑되지 않은 **새 화면**을 식별
3. 새 화면에 대해서만 컴포넌트 명세, 상태 시각 변화, 접근성을 추출
4. 기존 화면의 UI 명세는 원문 그대로 보존 — 내용 변경 금지
5. 새 화면에서 기존 컴포넌트를 재사용하면 참조만 추가 (정의 중복 금지)
6. 기존 화면에 새 요소가 추가된 경우에만 해당 화면 섹션을 최소 수정

## 허용: 토큰 참조, 컴포넌트 시각 명세, 인터랙션, 반응형, 접근성
## 금지: 비즈니스 로직, 상태 관리 구현, API 호출 방식

## 추출 전략

1. WF의 모든 UI 요소 나열 → 컴포넌트 후보
2. 디자이너 산출물에서 시각 속성 매핑
3. 속성을 디자인 시스템 토큰으로 정규화
4. WF 상태 매트릭스 각 상태에 시각 변화 정의
5. 접근성 요구사항 추가

## 품질 자가 점검

- [ ] WF 요소 ↔ 컴포넌트 매핑 = 100%
- [ ] 하드코딩 값 = 0건
- [ ] 모든 상태 정의됨
- [ ] 접근성 필수 항목 정의됨

## 출력 위치: specs/UI-{YYYY}-{NNN}.md
