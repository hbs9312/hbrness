# 와이어프레임 검증 규칙

## 구조 완전성 (critical)
1. 메타데이터 + 기능 참조 존재
2. 필수 3개 섹션 (전환 맵, 와이어프레임, 상태 매트릭스)
3. FS UI/UX 화면 수 = WF 화면 수

## 화면 전환 맵 (critical)
4. 정방향 경로 완전성
5. 역방향(뒤로가기) 경로 완전성
6. 조건부 분기 명시

## 와이어프레임 품질 (warning → critical)
7. 요소 설명 존재
8. 인터랙션 노트 (활성/비활성 조건) → 없으면 critical 승격
9. BR → UI 요소 매핑

## 상태 매트릭스 (critical)
10. 매트릭스 존재
11. 빈 셀 0 (해당 없음 표기 허용)
12. 에러 상태에 복구 경로 포함

## 경계 검증 (warning)
13. 디자인 침범: hex 색상, px 크기, 애니메이션 ms, 토큰명, 간격 토큰

## 크로스 참조 정확성 (warning)
14. 위치 기반 대명사 탐지: 전환 맵/와이어프레임/상태 매트릭스 본문에서 "위 {명사}", "상기 {명사}", "해당 {명사}" 등이 **다른 화면/요소/BR**을 가리키는 경우 → 정확한 이름·ID(`{화면명}`, `BR-NNN`) 사용 권고. `conventions.md#크로스-참조-규칙` 참조.
    - 예외: 같은 화면/블록 내부의 인접 bullet 목록을 "아래/위 항목"으로 가리키는 경우는 허용.
