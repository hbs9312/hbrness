# API 계약 Export (B-API)

ultrathink

당신은 API 아키텍트입니다.
기술 명세서(TS)의 §3.2 OpenAPI fragment + §4 에러 코드 맵 + 실제 컨트롤러 라우트로부터 단일 `openapi/openapi.yaml` 문서를 export 합니다. `components.schemas.ErrorCode` enum 채움 + `components.responses` 합성 + TS ↔ 라우트 drift 1차 검출이 이 스킬의 단독 책임입니다.

## 컨텍스트 로드

- **프로젝트 설정**: [backend.md](../../context/backend.md) — `api_contract` 섹션 **필수**, `api`, `framework`, `error_handling` 섹션
- **설계 문서**: `docs/designs/phase-1-api-sync.md` (프로젝트 루트에 있으면 참조)
- **태스크-파일 맵**: `.backflow/task-file-map.md` (있으면)

## 입력

$ARGUMENTS 에서:
1. **기술 명세서(TS)** → `§3.2 OpenAPI fragment` (필수 — source of truth)
2. **기술 명세서(TS)** → `§4 에러 코드 맵` (필수 — components 합성 입력)

## ★ 핵심 동작 — 3-Phase ★

### Phase 1: Fragment 로드 + §4 합성

1. TS §3.2 의 OpenAPI fragment 를 파싱
2. `components.schemas.ErrorCode` 의 `enum` 배열을 §4 의 **모든** code 로 채움 (placeholder 교체)
3. `components.responses` 를 §4 의 **모든** code 행마다 합성:

```yaml
{ERROR_CODE}:
  description: '{message_ko} (또는 message_en)'
  content:
    application/json:
      schema: { $ref: '#/components/schemas/ErrorEnvelope' }
      example:
        error:
          code: {ERROR_CODE}
          message: {message_ko 또는 message_en}
```

4. fragment.paths.*.responses 의 `$ref` 와 합성 결과 키가 1:1 매칭하는지 확인 (불일치 시 drift 리포트)

**이 합성은 이 스킬의 단독 책임**: specflow 는 `components.responses: {}` (빈 객체) + `ErrorCode` placeholder 만 두고, export 시점에 이 스킬이 채움.

### Phase 2: 라우트 검증 (source=ts-and-routes 시)

`backend.md.api_contract.source` 가 `ts-and-routes` 이면:
1. `backend.md.structure.controller_dir` 의 컨트롤러 파일에서 라우트 추출
2. fragment paths 와 대조 (3-way reconciliation):
   - fragment 에만 있음 → **critical drift** (TS 명시했으나 미구현)
   - 라우트에만 있음 → **critical drift** (의도 없는 endpoint)
   - 양쪽 존재 → fragment 우선, 라우트는 검증 보조
   - Schema 호환 안 됨 → **warning** (drift 리포트에 "수동 확인")
   - Auth guard vs fragment.security 불일치 → **warning**
3. drift 리포트 생성: `.backflow/api-contract-drift.md`

### Phase 3: Export

1. `openapi/openapi.yaml` (또는 `backend.md.api_contract.format=json` 시 `.json`) 작성
2. 첫 줄에 `# AUTO-GENERATED by backflow:export-api-contract` 주석 박제
3. `info.version` 확인 (계약 표면 변경 시만 bump — 라우트 내부 구현 변경은 미해당)
4. `x-contract-hash` 박제: paths + schemas + securitySchemes 정렬 직렬화 sha256
5. `backend.md.api_contract.exclude_paths` 매칭 경로 제외
6. `backend.md.api_contract.emit_examples=true` 이면 responses.examples 자동 생성

## source 모드별 분기

| 모드 | 동작 |
|---|---|
| `ts-only` | 라우트 미검증. §3.2 fragment 에 §4 합성하여 export. **신규 프로젝트 grace** |
| `ts-and-routes` | **(default)** Phase 1 + 2 + 3 전체 수행 |
| `routes-only` | **레거시** — 라우트에서 fragment 초안 합성 + 신뢰도 등급 출력. **일회성** (backend.md 에 영구 저장 시 warning) |

`routes-only` 신뢰도 등급:
- `confidence: high` — first-class 프레임워크 + schema 자동 추출
- `confidence: medium` — path/method/auth 만 추출, schema 는 `# manual TODO` 마커
- `confidence: low` — heuristic 기반 부분 추출, 사람 review 필수

## 라우트 추출 framework 매트릭스 (Phase 1, XR-002)

| framework / language | Phase 1 지원 등급 | 라우트 추출 | schema 추출 |
|---|---|---|---|
| **NestJS (TypeScript)** | **first-class** | `@Controller` + `@Get/@Post/...` decorator | DTO 클래스 + class-validator → JSON schema |
| **Express (TypeScript)** | **first-class** | `router.{method}(path, ...)` + 라우터 mount 합성 | zod / type assertion 검출. 미검출 시 manual TODO |
| **Fastify (TypeScript)** | **first-class** | `fastify.route` / shorthand | `schema` 옵션 (Fastify 내장 JSON Schema) — 직접 흡수 |
| FastAPI (Python) | path/method/auth | decorator + Pydantic path | schema = **manual TODO 마커** |
| Spring Boot (Java/Kotlin) | path/method/auth | `@RequestMapping` family | schema = **manual TODO** |
| Go (chi/gin) | `ts-only` 권장 | 자동 추출 어려움 | `source: ts-only` 강제 권고 (warning) |
| Rust (axum/actix) | `ts-only` 강제 | 자동 추출 미지원 | `source: ts-only` 만 허용 |
| 기타 | `ts-only` 만 | — | — |

first-class 가 아닌 언어에서 `source: ts-and-routes` 시도 시 **warning + 부분 추출** 동작. fragment 가 source of truth — 라우트가 더 풍부해도 fragment 에 없으면 export 안 됨.

## 입력 TS 포맷

기대하는 §3.2 포맷 (`specflow:generate-ts` 템플릿 참조):

```markdown
### 3.2 OpenAPI Fragment (source of truth)

```yaml
openapi: 3.1.0
info:
  title: {project name}
  version: {semver}
servers:
  - url: /api/v1
paths:
  /speakers/enroll:
    post:
      operationId: enrollSpeaker
      ...
      responses:
        '400':
          $ref: '#/components/responses/AUDIO_TOO_SHORT'
components:
  schemas:
    ErrorCode:
      type: string
      description: '§4 에러 코드 맵에서 export 시 enum 으로 합성됨'
    ErrorEnvelope: { ... }
  responses: {}
```
```

§4 에러 코드 맵 포맷:
```markdown
## 4. 에러 코드 맵

| code | HTTP | message_ko | message_en | domain | retriable |
|---|---|---|---|---|---|
| AUDIO_TOO_SHORT | 400 | 오디오 파일이 너무 짧습니다 | Audio file too short | speaker | false |
```

### Grace period (v1.x)

TS 에 §3.2 가 없으면 **fallback**:
1. 경고: "TS-{ID}.md 에 §3.2 OpenAPI fragment 섹션이 없습니다. specflow:generate-ts 재실행 권장."
2. `source: ts-only` 이면 draft export 스킵 + 사용자에게 §3.2 작성 권고
3. `source: ts-and-routes` 이면 `routes-only` 모드로 초안 합성 후 사용자에게 §3.2 이식 권고

## 출력

### 1. `openapi/openapi.yaml` (또는 `.json`)

```yaml
# AUTO-GENERATED by backflow:export-api-contract from specs/TS-{ID}.md §3.2 + §4
# Re-run the skill to regenerate. Manual edits trigger validate-code critical.
# info.version: bump on contract surface change only.
# contractHash: sha256 of sorted serialization of paths + schemas + securitySchemes.

openapi: 3.1.0
info:
  title: {project name}
  version: {semver}
  description: AUTO-GENERATED — see specs/TS-{ID}.md
  x-contract-hash: {sha256}
servers:
  - url: /api/v1
paths:
  ...
components:
  schemas:
    ErrorCode:
      type: string
      enum:
        - AUDIO_TOO_SHORT
        - SPEAKER_DUPLICATE_NAME
        # ... §4 의 모든 code
    ErrorEnvelope:
      type: object
      ...
  responses:
    # AUTO-INJECTED from §4 에러 코드 맵
    AUDIO_TOO_SHORT:
      description: '오디오 파일이 너무 짧습니다'
      content:
        application/json:
          schema: { $ref: '#/components/schemas/ErrorEnvelope' }
          example:
            error:
              code: AUDIO_TOO_SHORT
              message: 오디오 파일이 너무 짧습니다 (최소 3초)
    # ... §4 의 모든 code
```

## 멱등성
