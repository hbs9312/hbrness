# Test Specification (QA) Review Preset

## Default Context

이 파일은 테스트 명세서(QA Specification)입니다. 기능 명세(FS)·기술 명세(TS)·와이어프레임(WF)을 기반으로 작성된 테스트 케이스 정의로, QA·개발자가 이를 실행하여 릴리스 품질을 보증합니다.

## Review Perspective

### 1. 테스트 커버리지 (Test Coverage)
- FS의 모든 수락 기준마다 대응하는 테스트 케이스가 존재하는가
- TS의 모든 엔드포인트·에러 코드가 테스트되는가
- WF의 모든 상태 전이가 테스트되는가
- 회귀 테스트 범위가 정의되는가

### 2. 엣지 케이스 (Edge Cases)
- 경계값(최소·최대·0·빈 값)이 테스트되는가
- 이상 입력(형식 오류·너무 긴 값·특수문자)이 커버되는가
- 에러 경로(네트워크 실패·서버 오류·권한 부족)가 포함되는가
- 동시성 시나리오(중복 클릭·동시 요청)가 있는가

### 3. 테스트 가능성 (Testability)
- Given-When-Then 구조가 명확한가
- 전제 조건(테스트 데이터·사용자 상태)이 재현 가능한가
- 기대 결과가 검증 가능하게 구체적인가 (예: "정상 동작" X → "HTTP 200 + id 반환" O)
- 테스트 간 의존성 없이 독립 실행 가능한가

### 4. 우선순위·분류 (Priority & Category)
- 스모크·회귀·상세 테스트 분류가 명시되는가
- 우선순위(P0~P2 등)가 부여되는가
- 자동화 가능/수동 구분이 있는가
- 성능·보안·접근성 등 비기능 테스트 포함 여부

### 5. FS/TS/WF와의 일치 (Cross-Doc Alignment)
- 참조하는 수락 기준·엔드포인트·화면 ID가 명시되는가
- 상위 문서 변경 시 업데이트가 반영되었는가
- 상위 문서에는 있지만 테스트에 없는 항목, 또는 반대 사례

### 6. 테스트 환경·데이터
- 테스트 환경(dev·stage·prod 시뮬레이션) 구분
- 시드 데이터·Mock·Stub 정책이 명시되는가
- 외부 의존성 테스트 전략(실서비스 vs 가상)

## Severity Guidance

- **critical**: FS 수락 기준에 대응하는 테스트 누락, 에러 경로 미테스트, 보안·결제 관련 테스트 부재
- **warning**: 엣지 케이스 누락, 기대 결과 모호, 의존성 정책 불명확, 우선순위 미지정
- **info**: 테스트 최적화 제안, 자동화 권고, 구조 개선
