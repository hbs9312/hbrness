# Feature Specification (FS) Review Preset

## Default Context

이 파일은 기능 명세서(Feature Specification, FS)입니다. PM이 작성한 문서로, 개발자가 이를 기반으로 기술 명세(TS)·와이어프레임(WF)·구현 코드를 만듭니다. 문서 자체만 보고 구현 가능 수준인지, 누락·모호함이 없는지 평가합니다.

## Review Perspective

### 1. 명확성 (Clarity)
- 각 요구사항이 모호하지 않게 서술되었는가
- 용어가 문서 전반에서 일관되게 사용되는가 (용어집 필요 시 포함 여부)
- 주체(누가)·대상(무엇을)·조건(언제)·결과(어떻게)가 분명한가

### 2. 완전성 (Completeness)
- 해피 패스뿐 아니라 엣지 케이스·에러 케이스가 다뤄졌는가
- 인증·인가 조건(로그인 여부, 권한, 역할)이 명시되었는가
- 데이터 유효성 규칙(길이, 형식, 범위)이 정의되었는가
- 빈 상태·로딩 상태·실패 상태가 다뤄지는가
- 외부 의존성(결제·인증·알림 등)의 실패 처리가 서술되었는가

### 3. 테스트 가능성 (Testability)
- 각 요구사항이 검증 가능한 수락 기준(Acceptance Criteria)을 가지는가
- 측정 가능한 성공 지표가 있는가
- "사용자 경험이 좋아야 한다" 같은 검증 불가 문구가 있는가

### 4. 일관성 (Consistency)
- 동일 개념이 문서 내에서 같은 표현으로 쓰이는가
- 요구사항 간 충돌이 없는가 (예: 한 곳은 필수, 다른 곳은 선택)
- 참조되는 다른 문서·화면과의 정합성이 보장되는가

### 5. 스코프 (Scope)
- 과다 스펙: 이 릴리스에서 구현하지 않을 것이 포함됨
- 과소 스펙: 논리적으로 반드시 필요한 기능이 누락됨
- "추후에" "나중에" 등 미해결 지연 항목이 명시되지 않음

### 6. 비기능 요구사항
- 성능·가용성·접근성·보안 요구사항이 명시되었는가
- 데이터 보관·삭제·개인정보 처리 규정이 있는가

## Severity Guidance

- **critical**: 구현 불가능한 수준의 모호함, 보안·개인정보·데이터 손실 관련 누락, 요구사항 간 충돌
- **warning**: 엣지 케이스 미정의, 수락 기준 부재, 외부 의존성 실패 경로 누락
- **info**: 표현 개선, 구조 권고, 용어 일관성 제안
