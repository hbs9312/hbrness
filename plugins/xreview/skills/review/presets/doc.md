# General Document Review Preset

## Default Context

이 파일은 일반 문서(README·PRD·기술 글·설명서 등)입니다. 특정 전문 영역(코드·명세·디자인)으로 분류되지 않거나 여러 성격이 혼합된 경우 이 프리셋이 사용됩니다. 독자에게 전달하려는 메시지가 명확하고 충분한지를 평가합니다.

## Review Perspective

### 1. 명확성 (Clarity)
- 의도한 독자가 누구인지 드러나는가
- 전제 조건(배경 지식·용어)이 명시되는가
- 모호하거나 중의적 표현이 있는가
- 핵심 메시지가 도입부에서 전달되는가

### 2. 완전성 (Completeness)
- 의도한 범위 내 주요 내용이 모두 포함되었는가
- 예시·근거·수치가 필요한 곳에 제공되는가
- "추후에"·"TBD"로 미해결인 부분이 명시되는가

### 3. 정확성 (Accuracy)
- 사실 오류·잘못된 참조가 있는가
- 링크·인용이 유효하고 최신인가
- 코드·명령·경로 예시가 실제 동작하는가
- 버전·날짜 정보가 최신인가

### 4. 구조 (Structure)
- 목차·섹션 구성이 논리적인가
- 각 섹션이 한 가지 주제에 집중하는가
- 스캐닝 가능한 서술(짧은 단락·목록·헤딩) 구성
- 길이가 목적에 비해 과다/과소하지 않은가

### 5. 실행 가능성 (Actionability)
- 독자가 이 문서를 읽고 무엇을 해야 하는지 알 수 있는가
- 설치·실행·적용 절차가 따라하기 쉬운가
- 후속 자료·참고 링크가 제공되는가

### 6. 톤·일관성 (Tone & Consistency)
- 문체(경어·평어·기술적 톤)가 일관되는가
- 용어 사용이 통일되는가
- 약어·영문 표기·숫자 형식이 일관되는가

### 7. 가독성 (Readability)
- 문장 길이·복잡도가 적절한가
- 전문 용어 남용 없이 필요시 주석·용어집 제공
- 시각적 보조(표·코드 블록·다이어그램) 활용

## Severity Guidance

- **critical**: 사실 오류, 중요 정보 누락, 작동하지 않는 예시, 잘못된 지침
- **warning**: 모호함, 구조적 혼란, 일관성 결여, 후속 경로 부재
- **info**: 표현 개선, 추가 예시 권고, 가독성 다듬기
