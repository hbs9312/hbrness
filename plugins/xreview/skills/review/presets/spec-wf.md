# Wireframe (WF) Review Preset

## Default Context

이 파일은 와이어프레임(Wireframe, WF)입니다. 기능 명세(FS)를 기반으로 화면 구조와 상태 흐름을 저충실도로 표현한 문서입니다. 디자이너·개발자가 이를 보고 UI 명세와 구현을 만듭니다.

## Review Perspective

### 1. 화면 커버리지 (Screen Coverage)
- FS의 모든 기능이 하나 이상의 화면으로 표현되는가
- 진입 지점·종료 지점이 명확한가
- 화면 간 네비게이션 경로가 정의되는가

### 2. 상태 매트릭스 (State Matrix)
각 화면에 대해 다음 상태가 다뤄지는가:
- 초기(initial) / 로딩(loading)
- 비어있음(empty) / 데이터 없음
- 정상(populated)
- 에러(error) / 네트워크 실패 / 권한 부족
- 부분 성공 / 진행 중 / 완료

### 3. 사용자 흐름 (User Flow)
- 주요 플로우(등록·로그인·구매 등)가 처음부터 끝까지 연결되는가
- 분기(조건부 흐름)가 명시되는가
- 이탈·취소 경로가 정의되는가
- 뒤로가기·새로고침·중단 시 동작

### 4. 상호작용 (Interactions)
- 버튼·입력 요소의 트리거와 결과가 명시되는가
- 모달·토스트·알림의 발생 조건과 해제 조건이 있는가
- 키보드·터치·스와이프 등 대체 상호작용이 고려되는가

### 5. FS와의 정합성 (FS Alignment)
- FS의 수락 기준이 화면상에서 달성 가능한가
- FS의 데이터 유효성 규칙이 입력 화면에 반영되는가
- FS의 권한 조건이 화면 노출 규칙에 반영되는가

### 6. 접근성·반응형 고려
- 주요 브레이크포인트(데스크톱·태블릿·모바일)가 고려되는가
- 대형 컨텐츠·스크롤 영역의 처리
- 포커스 순서·탭 이동

## Severity Guidance

- **critical**: 주요 플로우 누락, 에러·빈 상태 미정의, FS 수락 기준 미달, 이탈 경로 없음
- **warning**: 일부 상태 누락, 상호작용 결과 모호, 반응형 고려 부재
- **info**: 흐름 최적화 제안, 대체 상호작용 권고
