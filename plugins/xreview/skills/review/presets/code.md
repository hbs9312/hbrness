# Code Review Preset

## Default Context

이 파일들은 구현된 소스 코드입니다. 기능 요구사항이나 기술 명세를 기반으로 작성되었으며, 프로덕션 환경에 배포될 수 있습니다. 동료 엔지니어가 머지 여부를 판단하기 위한 엄격한 리뷰가 필요합니다.

## Review Perspective

다음 관점을 순서대로 점검하세요. 각 항목에 해당하는 문제가 있으면 구체적인 파일/라인과 함께 finding으로 보고합니다.

### 1. 정확성 (Correctness)
- 비즈니스 로직이 의도한 결과를 생성하는가
- 엣지 케이스(빈 값, 경계값, 비정상 입력)를 처리하는가
- 조건 분기의 누락/역전이 있는가
- 타입·null 처리가 올바른가

### 2. 보안 (Security)
- 인증·인가 검증이 엔드포인트·작업 단위로 수행되는가
- 사용자 입력 검증·이스케이핑(SQL/XSS/Command Injection 등)
- 민감 정보(토큰·비밀번호·PII)의 로깅·응답 노출
- 권한 상승, 수평 이동, IDOR 가능성

### 3. 에러 처리 (Error Handling)
- 외부 의존성(네트워크·DB·큐) 실패 시 복구 경로
- 예외 전파가 올바른 경계에서 일어나는가
- 에러 메시지가 내부 구조를 노출하지 않는가
- 멱등성(idempotency)·재시도·트랜잭션 경계

### 4. 동시성·일관성 (Concurrency & Consistency)
- 경쟁 조건, 데드락, 순서 가정
- 트랜잭션 범위와 격리 수준
- 캐시·영속성 사이의 일관성

### 5. 성능 (Performance)
- N+1 쿼리, 불필요한 반복, 메모리 누수
- 부적절한 동기 I/O, 블로킹 호출
- 큰 입력·트래픽에서의 동작

### 6. 계약 일치 (Contract Adherence)
- 명세서(TS/FS)가 함께 제공되면 그와의 일치 여부
- 공개 API 시그니처·스키마·상태 코드·에러 코드
- DB 스키마 제약과 코드상의 가정 일치

### 7. 유지보수성 (Maintainability)
- 네이밍·구조의 명확성
- 순환 복잡도, 중복 코드
- 의존 방향·레이어링 위반
- 테스트 가능성

## Severity Guidance

- **critical**: 보안 취약점, 데이터 손실·손상, 기능 오류, 계약 위반, 프로덕션 중단 가능성
- **warning**: 성능 이슈, 잠재적 버그, 에러 처리 누락, 동시성 미정의, 유지보수 저하
- **info**: 스타일·네이밍 개선 제안, 리팩터링 아이디어, 대안 구현 제시
