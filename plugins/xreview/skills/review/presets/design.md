# Design Review Preset

## Default Context

이 파일은 디자인 산출물(Figma 파일·이미지 파일 등)입니다. 디자이너가 UI 명세 또는 와이어프레임의 의도를 시각적으로 구현한 것으로, 디자인 시스템·접근성·UX 관점에서 평가합니다.

주의: 이미지·Figma 파일은 직접 읽기가 제한될 수 있습니다. 파일 경로·URL만 전달된 경우 외부 LLM에 메타데이터와 함께 설명적 리뷰를 요청하세요. 이미지 수신이 가능한 모델(예: GPT-5.4 멀티모달)을 권장합니다.

## Review Perspective

### 1. 디자인 시스템 준수 (Design System Adherence)
- 색상·타이포·간격·그림자·반경이 토큰 체계를 따르는가
- 기존 컴포넌트를 재사용하는가 vs 신규 변형이 남발되는가
- 브랜드·제품 가이드라인(로고 마진·색 사용 규칙)을 준수하는가

### 2. 일관성 (Consistency)
- 유사한 기능·요소가 일관된 시각 언어로 표현되는가
- 버튼·인풋·카드 등 동일 타입의 스타일이 통일되는가
- 전체 화면 간 시각적 리듬이 유지되는가

### 3. 정보 위계 (Information Hierarchy)
- 가장 중요한 정보·액션이 시각적으로 두드러지는가
- 타이포 스케일이 계층 구조를 표현하는가
- 공간·그룹핑으로 관계가 드러나는가

### 4. 사용자 경험 (UX)
- 시선 흐름이 자연스럽고 주요 액션으로 유도되는가
- 마찰 요소(불필요한 스텝·선택지 과다) 여부
- 피드백(성공·실패·진행)이 충분히 표현되는가
- 에러·빈 상태의 감정적 톤이 적절한가

### 5. 접근성 (Accessibility)
- 색 대비(배경-전경, 텍스트-텍스트) 충분한가 — WCAG AA 이상
- 색 의존적 정보 전달이 없는가 (색 외에 아이콘·텍스트 병행)
- 터치 타깃 크기(≥ 44×44px)
- 포커스 가능 요소의 시각적 구별

### 6. 반응형·적응형
- 브레이크포인트별 레이아웃 변화가 자연스러운가
- 컨텐츠 가변성(긴 텍스트·많은 항목) 대응
- 터치·마우스 환경 차이 고려

### 7. WF·UI 명세와의 일치
- 와이어프레임의 의도(구조·기능·흐름)가 유지되는가
- UI 명세의 컴포넌트 범주·상태가 반영되는가

### 8. 구현 가능성 (Implementability)
- 토큰·컴포넌트 체계로 실현 가능한 디자인인가
- 과도한 커스텀이 유지보수 부담을 만들지 않는가
- 애니메이션·효과의 기술적 실현 가능성

## Severity Guidance

- **critical**: 접근성 기준 실패(대비·포커스·터치 타깃), 브랜드 가이드 위반, WF 위반
- **warning**: 디자인 시스템 벗어난 컴포넌트, 토큰 미사용, 반응형 누락, 정보 위계 약함
- **info**: 시각적 개선 제안, 마이크로 인터랙션 권고, 일관성 다듬기
